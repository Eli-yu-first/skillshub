# 软件总体设计说明书：AI技能中心平台 V1.0

## 1. 引言

本说明书旨在详细阐述“AI技能中心平台 V1.0”（SkillsHub）的总体设计方案。文档内容涵盖了系统的核心架构、模块划分、数据流程、技术特点、运行环境及性能指标，为软件的后续开发、测试、运维及版本迭代提供明确的技术框架和指导原则。本设计旨在构建一个技术先进、类型安全、高度可扩展的AI技能协作与分享平台。

## 2. 系统架构

本软件采用基于 **Next.js** 的全栈开发框架，这是一种集成了前端渲染与后端API功能的“一体化”架构。通过利用Next.js的 **App Router** 和 **Server Actions**，系统在单个代码库中实现了高效的前后端协作和数据交互，同时保证了优秀的服务端渲染（SSR）和静态站点生成（SSG）能力。整体架构可分为客户端层、应用服务层（由Next.js统一管理）和数据存储层。

### 2.1. 架构图

```
+--------------------------------------------------------------------+
|                            客户端层 (Client)                         |

|--------------------------------------------------------------------|

|---------------------------------v----------------------------------|
```

### 2.2. 架构说明

- **客户端与应用服务层的融合**：本系统架构的核心是 **Next.js**。Next.js打破了传统前后端分离的物理界限，允许开发者在同一个项目中编写前端UI和后端逻辑。
    - **React Server Components (RSC)**：大部分页面和组件默认在服务器端渲染，它们可以直接访问数据库或执行后端逻辑，然后将渲染好的HTML发送到客户端。这极大地提升了首屏加载速度和SEO效果。
    - **Client Components**：对于需要用户交互（如点击事件、状态管理）的组件，可以通过添加`'use client'`指令，使其在客户端执行和渲染。
    - **Server Actions**：这是一种革命性的数据交互方式。前端组件可以直接调用在后端定义的异步函数（Server Action），而无需手动编写API路由和请求代码。Next.js会自动处理其中的网络通信，并保证类型安全。这被广泛用于处理表单提交、数据创建和更新等操作。
    - **API Routes**：对于需要被外部服务调用或更传统的API场景，Next.js也支持在`app/api`目录下创建标准的RESTful API端点。

- **数据访问层**：采用 **Drizzle ORM**。Drizzle是一个类型安全的SQL查询构建器，它通过分析在`server/db/schema.ts`中定义的数据库Schema来提供强类型的查询接口。这使得所有数据库操作都在TypeScript的类型检查保护之下，有效防止了SQL注入，并提升了代码质量。

- **数据存储层**：选用标准的关系型数据库（如 **MySQL** 或 **PostgreSQL**），以其强大的事务性、数据一致性和成熟的生态系统，为平台提供可靠的数据存储基础。

## 3. 模块划分

系统根据Next.js App Router的目录结构进行功能模块划分，每个目录通常对应一个URL路由，并包含该功能所需的所有组件和逻辑。

| 模块名称 (目录) | 核心功能 | 主要技术/文件 |
| :--- | :--- | :--- |
| **技能市场 (`app/marketplace`)** | 浏览、搜索、筛选所有公开技能 | `page.tsx` (Server Component) |
| **技能详情 (`app/skill/[id]`)** | 展示技能文档、提供演练场、查看版本 | `page.tsx`, `readme.tsx`, `playground.tsx` |
| **技能创建/编辑 (`app/creator`)** | 提供在线表单和代码编辑器用于创建和修改技能 | `page.tsx` (Client Component), Server Actions |
| **用户中心 (`app/dashboard`)** | 管理个人技能、收藏、团队和账户设置 | `layout.tsx`, `my-skills/page.tsx` |
| **认证模块 (`app/auth`)** | 处理用户登录、注册、会话管理 | NextAuth.js, `signIn/page.tsx` |
| **后端服务 (`server/`)** | 包含数据库Schema、Drizzle客户端和核心业务逻辑 | `db/schema.ts`, `actions/skill.ts` |

## 4. 数据流程

以“创建一个新技能”为例，说明系统的数据流转过程：

1.  **数据输入**：用户在客户端的技能创建页面（`app/creator/page.tsx`）填写表单，包括技能名称、描述以及在代码编辑器中编写的技能逻辑。
2.  **Server Action调用**：用户点击“发布”按钮，触发表单的`onSubmit`事件。该事件直接调用一个从后端文件（如`server/actions/skill.ts`）导入的名为 `createSkill` 的Server Action函数，并将表单数据作为参数传递。
3.  **后端处理**：
    -   `createSkill` 函数在服务器端执行。
    -   它首先对传入的数据进行校验，并验证用户是否已登录且有权限创建技能。
    -   接着，它调用Drizzle ORM的`insert`方法，将新的技能元数据写入数据库的`skills`表，并将技能代码和README存入`skill_versions`表。
4.  **响应与重定向**：
    -   数据库操作成功后，`createSkill` 函数可能会调用Next.js的`revalidatePath`函数来清除相关页面的缓存，确保技能市场的列表会更新。
    -   最后，它调用`redirect`函数，将用户重定向到新创建技能的详情页面。
5.  **前端更新**：浏览器接收到重定向指令，自动跳转到新的URL，用户即可看到刚刚创建的技能详情。整个过程无需手写任何`fetch`或`axios`代码，也无需管理加载状态，由Next.js框架自动完成。

## 5. 技术特点

- **全栈类型安全**：通过 **Next.js + TypeScript + Drizzle ORM** 的组合，实现了从数据库到后端逻辑再到前端UI的端到端类型安全，是本系统最核心的技术优势。
- **极致的开发体验**：**Server Actions** 和 **React Server Components** 大幅简化了数据获取和提交的逻辑，让开发者能更专注于业务本身，显著提升了开发效率。
- **高性能与SEO友好**：默认的服务端渲染（SSR）确保了用户能够快速看到页面内容，同时也对搜索引擎优化（SEO）非常友好，这对于一个内容驱动的平台至关重要。
- **灵活的渲染策略**：Next.js允许在页面或组件级别灵活选择渲染策略（SSR, SSG, CSR），使得系统可以根据不同页面的特点进行性能优化。

## 6. 运行环境

- **服务器端/开发环境**：
    -   操作系统：Linux, Windows, macOS
    -   运行环境：Node.js 18 或更高版本
    -   数据库：MySQL 8.0+ 或 PostgreSQL 14+
- **客户端**：
    -   支持ECMAScript 6的现代桌面或移动浏览器。

## 7. 性能特点

- **快速的首屏加载**：得益于服务端渲染，用户请求页面时，服务器直接返回完整的HTML，浏览器无需等待JS加载和执行即可渲染内容。
- **高效的数据交互**：Server Actions通过RPC（远程过程调用）的方式进行通信，相比传统的RESTful API，其数据传输更高效，且能更好地进行批量处理。
- **自动代码分割**：Next.js App Router会自动根据页面路由对代码进行分割，用户访问某个页面时，只需加载该页面所需的最小代码量。
- **可扩展的基础设施**：部署在Vercel等现代托管平台时，Next.js应用可以轻松实现全球CDN分发、自动扩缩容和边缘计算，具备优秀的水平扩展能力。
